# 마이링크 (MyLink) 사용자 시나리오

## 1. 방문자 시나리오 (Public Page Viewer)
**목표:** 특정 사용자의 마이링크 페이지를 방문하여 원하는 링크로 이동한다.

- 사용자는 **공유된 마이링크 주소(`mylink.com/username`)에 접속하기 위해** 브라우저 주소창이나 SNS 프로필 링크를 **클릭(또는 입력)한다.**
- 사용자는 **페이지 소유자가 누구인지 확인하기 위해** 상단의 프로필 사진, 닉네임(`displayName`), 소개글을 **열람한다.**
- 사용자는 **관심 있는 목적지로 이동하기 위해** 화면에 나열된 링크 버튼 목록과 각 링크의 파비콘 아이콘을 **살펴본다.**
- 사용자는 **특정 웹사이트나 콘텐츠로 이동하기 위해** 원하는 링크 버튼을 **클릭한다.**
- 사용자는 **새 탭에서 열린 목적지 페이지를 탐색하기 위해** 해당 웹사이트 메뉴를 **이용한다.**

*(예외 케이스)*
- 사용자는 **유효하지 않거나 삭제된 계정 주소로 잘못 접속한 경우, 올바른 주소를 다시 확인하기 위해** '해당 프로필을 찾을 수 없습니다' 안내 화면(404 에러뷰)을 **확인한다.**

---

## 2. 소유자 시나리오 (Link Owner / Admin)
**목표:** 자신의 마이링크 페이지를 관리(링크 추가, 수정, 순서 변경, 삭제)한다.

### 2.1 링크 추가 시나리오
- 사용자는 **자신의 마이링크 관리자 페이지에 접속하기 위해** 구글 계정으로 **로그인한다.**
- 사용자는 **새로운 링크 버튼을 만들기 위해** 화면 하단(또는 상단)의 '새 링크 추가' 버튼을 **클릭한다.**
- 사용자는 **추가할 링크의 목적지를 구체적으로 설정하기 위해** URL 입력칸에 실제 이동할 웹 주소를 **붙여넣기(입력)한다.**
- 사용자는 **방문자에게 이 링크가 어떤 링크인지 설명하기 위해** 제목(Title) 텍스트를 자유롭게 **입력한다.**
- 사용자는 **입력한 URL의 대상 웹사이트 파비콘(Favicon)이 아이콘으로 정상 추출되는지 확인하기 위해** 잠시 **대기한다.**
- 사용자는 **새로 만든 링크를 방문자에게 즉시 노출하기 위해** 활성화(ON) 상태인지 확인하고 변경사항을 **저장한다.**

### 2.2 링크 수정 및 순서 변경 시나리오
- 사용자는 **기존에 등록된 링크의 제목이나 URL을 변경하기 위해** 해당 링크 항목의 '수정' 아이콘(또는 텍스트 입력창)을 **클릭(포커스)한다.**
- 사용자는 **변동된 정보를 즉시 반영하기 위해** 텍스트를 수정하고 엔터를 치거나 다른 곳을 탭하여 **저장한다.**
- 사용자는 **가장 중요한 링크를 상단에 배치하기 위해** 링크 항목 측면의 핸들(드래그 손잡이)을 마우스 누른 채로 원하는 위치로 **드래그 앤 드롭(Drag & Drop)한다.**
- 사용자는 **특정 기간에만 노출할 링크를 잠시 숨기기 위해** 링크 우측의 활성화 스위치(Toggle)를 **OFF로 변경한다.**

### 2.3 링크 삭제 시나리오
- 사용자는 **더 이상 필요 없는 링크를 영구적으로 프로필에서 제거하기 위해** 해당 링크 항목의 '삭제' 버튼(휴지통 아이콘)을 **클릭한다.**
- 사용자는 **실수로 귀중한 링크를 삭제하는 것을 방지하기 위해** 나타나는 삭제 재확인 팝업창에서 '확인'을 최종적으로 **선택한다.**
