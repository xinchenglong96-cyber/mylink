# 마이링크 (MyLink) 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트명:** 마이링크 (MyLink)
- **목적:** 사용자가 자신의 여러 소셜 미디어, 포트폴리오, 웹사이트 등의 링크를 하나의 통합된 페이지(프로필)로 모아서 공유할 수 있는 서비스 제공
- **대상 사용자:** 인플루언서, 크리에이터, 프리랜서, 개인 포트폴리오가 필요한 사람 등 여러 링크를 효과적으로 알리고 싶은 모든 사용자

---

## 2. 핵심 기능 목록

### 필수 기능 (MVP - Minimum Viable Product)
1. **사용자 인증 (Firebase 기반 구글 로그인)**
2. **프로필 관리**
3. **링크 관리 시스템 (CRUD)**
4. **사용자 공유 페이지 (퍼블릭 뷰)**

### 제외 기능 (Out of Scope)
- 구글 로그인을 제외한 이메일/비밀번호 가입 및 기타 소셜 로그인 방식은 지원하지 않음

### 선택 기능 (*AI 제안 - 확장 및 개선점*)
이 부분은 기본적인 기능 외에 서비스의 질을 높이고 사용자 경험을 개선하기 위해 제안하는 기능입니다.
1. **커스텀 디자인 및 테마 적용 기능:** 배경, 버튼 스타일, 폰트 등을 사용자가 직접 꾸밀 수 있는 기능
2. **통계 및 애널리틱스 (Analytics):** 내 페이지 방문자 수 및 각 링크별 클릭 수 확인 기능
3. **소셜 미디어 전용 아이콘 연동:** 인스타그램, 유튜브, 깃허브 등 자주 쓰는 플랫폼에 맞는 UI 디자인 분리
4. **QR 코드 자동 생성:** 오프라인 명함이나 홍보물에 바로 사용할 수 있도록 프로필 주소 기반 QR코드 제공
5. **SEO 및 메타데이터 설정:** 프로필 링크를 카카오톡 등의 메신저나 SNS에 공유할 때 나타나는 썸네일과 설명 커스터마이징

---

## 3. 각 기능의 상세 설명

### 3.1 사용자 인증
- **Firebase Auth**를 활용한 **구글 소셜 로그인(Google Login)** 단일 방식만 지원
- 첫 로그인 시 화면을 전환하여 사용할 고유한 아이디(Username)를 설정하도록 유도
- 설정한 아이디는 퍼블릭 페이지 URL의 일부분(`mylink.com/아이디`)으로 사용됨

### 3.2 프로필 관리
- **프로필 사진:** 이미지 업로드 시 일정 비율(예: 원형)로 크롭하여 사용자 페이지 상단에 노출
- **닉네임 & 소개글:** 프로필 사진 아래에 노출될 이름과 간단한 한 줄 소개(Bio) 텍스트 입력 및 수정 가능

### 3.3 링크 관리 시스템 (CRUD)
- **링크 추가:** 사용자는 사용자 페이지에 표시될 버튼의 '제목(Title)'과 실제 이동할 '주소(URL)'를 입력하여 링크를 생성
- **파비콘(Favicon) 아이콘 자동 연동:** 입력된 링크 주소(URL)에서 대상 웹사이트의 파비콘을 추출해 해당 링크 아이템의 아이콘으로 자동 표시
- **순서 변경:** 등록된 링크 리스트에서 항목을 드래그 앤 드롭(Drag & Drop)하여 노출될 순서를 직관적으로 조절
- **상태 관리:** 특정 링크를 삭제하지 않고도 일시적으로 숨길 수 있도록 링크별 ON/OFF 활성화 토글 제공
- **수정 및 삭제:** 기존 링크의 텍스트 혹은 URL을 자유롭게 수정, 불필요한 링크 영구 삭제

### 3.4 사용자 공유 페이지 (퍼블릭 뷰)
- 해당 사용자의 고유 주소로 접속하면, 프로필 이미지・소개글・링크 버튼들이 모아져 있는 뷰 제공
- 모바일 가독성이 가장 중요하므로 모바일 디바이스에 최적화된 인터페이스(Mobile-First) 렌더링
- 이 페이지에서는 사용자의 링크 클릭 시 해당 URL로 새 창에서 이동하는 기능만 동작함

## 4. 데이터베이스 모델링 (Firebase Firestore)

Firebase Firestore의 Document-Subcollection 구조를 활용하여 모델링합니다.

### 4.1 `users` 컬렉션 (사용자 정보)
사용자별 최상위 데이터 문서이며, 1명당 1개의 Document를 가집니다. (Document ID: Firebase Auth의 `uid`)
- `uid` (String): Document ID와 동일
- `username` (String): 마이링크 공유 URL 경로로 사용될 사용자 고유 식별자 (중복 불가)
- `displayName` (String): 프로필에 나타날 사용자 닉네임 또는 활동명
- `photoURL` (String): 프로필 사진 이미지 URL
- `bio` (String): 프로필 하단의 짧은 소개글
- `createdAt` (Timestamp): 계정 생성 날짜
- `updatedAt` (Timestamp): 프로필 수정 날짜

### 4.2 `links` 서브 컬렉션 (사용자별 링크 관리)
각 사용자(`users` document) 내부에 위치한 하위 컬렉션으로, 사용자가 추가하는 개별 링크 항목들을 저장합니다.
- Document 경로: `users/{uid}/links/{linkId}`
  - `title` (String): 버튼에 표시될 링크 제목
  - `url` (String): 실제 이동할 도착 주소 (예: https://...)
  - `faviconUrl` (String): 입력된 URL에서 추출한 파비콘 이미지 주소
  - `order` (Number): 목록 내 정렬을 위한 순서 인덱스 값
  - `isActive` (Boolean): 사용자 뷰에서의 노출 여부 (ON/OFF 토글)
  - `createdAt` (Timestamp): 링크 등록 날짜
  - `updatedAt` (Timestamp): 링크 수정 날짜
